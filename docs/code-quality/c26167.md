# <a name="c26167"></a>C26167
upozornění C26167: pravděpodobně uvolňování zámku unheld \<Zámek > ve funkci \<func >.

 Upozornění C26167 vypadá takto: upozornění [C26117](../code-quality/c26117.md) s tím rozdílem, že hladina spolehlivosti je nižší. Například funkce může obsahovat chyby poznámky.

## <a name="example"></a>Příklad
 Následující kód vygeneruje C26167, jakož i C26110.

```cpp

typedef struct _DATA {
    CRITICAL_SECTION cs;
} DATA;

_Releases_lock_(p->cs) void Leave(DATA* p) {
    LeaveCriticalSection(&p->cs); // OK
}
void ReleaseUnheldLock(DATA* p) { // Warning C26167
    int i = 0;
    Leave(p); // Warning C26110
}

```

## <a name="example"></a>Příklad
 Následující kód opravíte tato upozornění.

```cpp

typedef struct _DATA {
    CRITICAL_SECTION cs;
} DATA;

_Releases_lock_(p->cs) void Leave(DATA* p) {
    LeaveCriticalSection( &p->cs );
}

void ReleaseUnheldLock( DATA* p ) {
    EnterCriticalSection( &p->cs );
    int i = 0;
    Leave(p);
}

```
